# SSH 隧道远程连接指南

如果你的 `Hermes Agent` 运行在远程服务器上，可以通过 SSH 隧道把本地桌面版和远程服务连接起来。

## 常见场景

- Agent 跑在 VPS、家用主机或办公室机器
- 你希望继续在本机 `Hermes Windos` 中操作
- 不想把远程服务直接裸露到公网

## 基本思路

1. 在远程机器上启动 Hermes Agent 服务
2. 通过 SSH 建立本地端口转发
3. 在桌面版里把接口地址指向本地转发端口

示例：

```bash
ssh -L 8765:127.0.0.1:8765 your-user@your-server
```

这表示把本机 `127.0.0.1:8765` 转发到远程机器的 `127.0.0.1:8765`。

## 在桌面版里如何填写

- Host / Base URL：`http://127.0.0.1:8765`
- 其余认证项按你的 Hermes Agent 服务配置填写

## 建议

- 优先使用 SSH 隧道，而不是直接暴露远程 Agent 端口
- 给远程服务加好访问控制
- 把常用连接方式写进你自己的部署文档
